# Power BI Dashboard Requirements

## Page 1: Executive Overview

Include total transaction value, transaction count, active customers, average transaction amount, approval rate, decline rate, potential anomaly count, monthly transaction trend, category value, channel volume, debit versus credit activity, and top customer segments.

## Page 2: Spending Analysis

Include spending by category, spending by segment, average spend per customer, monthly spending trend, weekday versus weekend activity, recurring versus nonrecurring transactions, top merchants, category share, and customer spending distribution.

## Page 3: Customer Analysis

Include active customers, transactions per customer, average customer spend, customer segment, age group, tenure, preferred channel, highest-activity customers, customer monthly behavior, and customer drill-through.

## Page 4: Channel and Merchant Analysis

Include transaction volume and value by channel, approval and decline rates by channel, merchant value and count, merchant category performance, top merchants, and channel usage by customer segment.

## Page 5: Transaction Status and Operations

Include approved, declined, pending, and reversed transactions; status trend over time; decline rate by category and channel; and transaction-processing summary.

## Page 6: Potentially Anomalous Activity

Label this page exactly as `Potentially Anomalous Activity`. Include total flagged transactions, flagged value, customers with flags, high-priority review count, flags by rule, flags by customer, flags by category, flags by channel, flags over time, a detail table, and a review-priority slicer.

Add this disclaimer: Flags are generated using analytical rules. A flag does not confirm fraud. Further investigation would be required.

## Page 7: Data Quality

Include total issues, critical issues, errors, warnings, issues by source, issues by validation rule, reconciliation status, duplicate transaction count, missing relationship count, and resolved versus unresolved issues.
